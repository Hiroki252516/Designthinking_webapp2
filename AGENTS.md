# AGENTS

## 目的
識別番号入力画面での遷移とエラーメッセージの挙動を統一する。

## 挙動要件 (入力画面)
- 確認ボタン/Enter 押下時に識別番号を検証する。
- 有効な番号の場合のみルーレット画面へ遷移する。
- 無効な番号の場合は **ルーレット画面へ遷移しない**。
- 無効時は入力画面に **「無効な番号です」** を表示する (文言は完全一致)。
- 入力内容が変更/クリアされたらエラーメッセージは消す。

## 実装メモ
- フロントは `frontend/index.html` / `frontend/app.js` を中心に対応する。
- エラーメッセージは `#input-error` を使い、表示/非表示を制御する。
- 確認ボタンは「4桁入力済み」で押下可能にし、押下時に有効/無効を判定する。
- もし API で有効性を判定する場合も **無効と判定されたら画面遷移しない**。
